1. Executive Summary
Vaakya is an autonomous, multi-agent AI platform that eliminates the legal document bottleneck for Indian small and medium businesses (SMBs). Powered by LangGraph, FastAPI, and Claude AI, Vaakya handles the entire legal document lifecycle - from a user typing a plain-English request or uploading a PDF, all the way to e-signature, obligation tracking, and dispute resolution - without any lawyer or legal team.

"Vaakya" (?????) is the Telugu word for a legal statement or clause - the smallest unit of law. The name reflects the project's soul: making every legal clause accessible and affordable to every Indian SMB, regardless of size or city.

Vaakya solves this with 8 specialized AI agents, a hybrid sequential-parallel LangGraph workflow, and zero dependency on expensive legal professionals for routine documents.

4. User Input Modes
Vaakya accepts user requests in two distinct modes. Both modes converge at the same LangGraph state and flow through the identical 8-agent pipeline after intake:

4.1 Mode 1 - Plain Text Request
The user types their requirement in plain English (or Telugu/Hindi) through the web dashboard or WhatsApp interface. The FastAPI endpoint POST /document/new receives a JSON body and the Arambha (Intake) Agent classifies the document type, extracts parties, jurisdiction, and key terms.

Example: "I need an NDA with TechCorp Pvt Ltd, Hyderabad. They should not share my source code or client data for 3 years. I want a Rs.5 lakh penalty clause and Andhra Pradesh jurisdiction."
4.2 Mode 2 - PDF / DOCX Upload
The user uploads an existing contract (received from a vendor, client, or lawyer). The FastAPI endpoint POST /document/upload accepts multipart/form-data. PyMuPDF (fitz) extracts text from every page with page-level labels. The extracted plain text then enters the same LangGraph state as Mode 1 - from the Arambha Agent onward, the pipeline is identical.

Example: "TechCorp sent me their NDA template. Can you review it and flag any risks before I sign?" - uploads TechCorp_NDA_draft.pdf

7. The Vaakya Workflow - Architecture
7.1 Hybrid Sequential + Parallel Design
Vaakya uses a Hybrid LangGraph Architecture - sequential where order matters, and parallel fan-out where agents are independent. This is the optimal design for legal document processing:

Sequential ZoneArambha -> Rachana -> Parisheelanam reflexion loop. Each step depends on the previous output - you cannot review before drafting, and cannot draft before classifying.Parallel Fan-out ZoneParisheelanam (final review) + Jokhim (risk flag) run simultaneously after the draft is ready. Both read the same finished document independently - perfect candidates for parallel execution.Parallel Redline ZoneSamjoota (negotiation) + Jokhim (risk scan) run in parallel on incoming redlined contracts. Independent agents, same document.HITL CheckpointHuman-in-the-loop pause before any document is sent for signature. The user reviews, can edit, and explicitly approves. LangGraph interrupt() handles this natively.Fan-in / MergeParallel branches write to list-append fields in the TypedDict state. A merge node collects both results before proceeding to HITL.
7.2 Sub-Graph A - New Document
1. User sends text request or uploads PDF to FastAPI endpoint
2. PyMuPDF extracts text if PDF - plain text enters LangGraph state
3. Arambha Agent: classify document type, extract parties, jurisdiction, key terms
4. Rachana Agent: generate full contract from clause library + Claude AI
5. Parisheelanam Agent: review draft, score risk 0-100
6. If risk score less than 75: loop back to Rachana (max 3 iterations)
7. PARALLEL FAN-OUT: Parisheelanam (final) + Jokhim run simultaneously
8. FAN-IN: merge results - HITL checkpoint (user reviews and approves)
9. Sahee Agent: PDF generation - Digio e-sign - encrypted S3 vault
10. Sruthi Agent: extract obligations - create WhatsApp alert calendar

7.3 Sub-Graph B - Redline Review
11. User uploads contract received from the other party
12. Arambha Agent: classify and identify as counter-party document
13. PARALLEL: Samjoota Agent + Jokhim Agent run simultaneously
14. Samjoota: diffs with standard terms, suggests counter-proposals
15. Jokhim: deep risk scan on incoming counter-party terms
16. FAN-IN - HITL checkpoint - Sahee Agent sends redlined version back

7.4 Sub-Graph C - Dispute Resolution
17. User raises a dispute flag against an existing signed contract
18. Arambha Agent: identifies relevant contract from vault by metadata
19. Vivada Agent: extracts applicable clauses, generates dispute summary
20. Vivada Agent: drafts legal notice under ICA / CPC as PDF
21. Sahee Agent: delivers legal notice via WhatsApp and email
